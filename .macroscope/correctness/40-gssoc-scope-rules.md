Apply these rules when reviewing contributor PRs, especially GSSoC work.

Scope review priorities:
- Check whether the PR fully solves the linked issue, not just a nearby symptom.
- Flag PRs that claim to close unrelated issues or include work from multiple issues without maintainer approval.
- Be strict about docs-only PRs staying docs-only, tests-only PRs staying tests-only, and single-issue UI fixes staying narrowly scoped.
- If a PR has no linked issue, ask for one and prefer a specific issue suggestion over a generic complaint.
- Keep feedback clear and actionable for beginners, but do not lower correctness standards.
